Database constraint violation: "properties_status_check"
Problem: The database enum only allows: 'pending', 'approved', 'rejected', 'inactive'
UI is sending: 'active', 'maintenance' (not in enum)
Solution: Add 'active' and 'maintenance' to the database enum

üîß Fix Database Enum

Warning: This will modify the database schema. Make sure you have a backup.

üìã SQL Commands to Execute

-- Add new enum values to property_status
ALTER TYPE property_status ADD VALUE IF NOT EXISTS 'active';
ALTER TYPE property_status ADD VALUE IF NOT EXISTS 'maintenance';

-- Verify the enum values
SELECT unnest(enum_range(NULL::property_status)) as status_values;

-- Update any invalid status values to 'pending'
UPDATE properties 
SET status = 'pending' 
WHERE status NOT IN ('pending', 'approved', 'rejected', 'inactive', 'active', 'maintenance');
